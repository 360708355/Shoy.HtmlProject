获取订单信息
1.接口说明
把商品加入购物车，期间会进行一系列判断，包括库存、活动、商品状态等。
2.应用场景
3.接口调用说明
3.1 请求说明
url http://www.api.100hg.com/apiTools/router.verify
格式 JSON
请求方式 GET
3.2 请求参数说明
名称 类型 是否必须 描述
method String 是 固定为：trade.orders.search
ticket String 是 用户令牌，包含登录后的用户信息，登录时获取。
order_type Int 否 订单类型：默认为0，全部；1为会员；12为商家
order_status String 是 订单状态,多个以,分割，若其中包含-10,则其他的无效：-10，全部；-2：未支付，-1：用户取消，0：未处理；1：正在备货；2：审核失败；3：已经发货；4：发货失败；5：交易完成。
3.3 返回参数说明
名称 类型 描述
state Int 状态：1成功，其他失败
msg String 错误信息
result Object 返回结果
result.count Int 订单数量
result.list Object 返回结果
result.list[i].order_num String 订单号
result.list[i].payment_type Int 支付方式，1：在线支付；3：余额支付。
result.list[i].order_state Int 订单状态，-2：未支付，-1：用户取消，0：未处理；1：正在备货；2：审核失败；3：已经发货；4：发货失败；5：交易完成。
result.list[i].products Array 商品列表
result.list[i].products[j].id Int 商品ID
result.list[i].products[j].num 商品编号
result.list[i].products[j].name String 商品名称
result.list[i].products[j].price Double 会员价
result.list[i].products[j].price_vip Double vip价，商家登录后，才能获取
result.list[i].products[j].count Int 该商品在购物车中的数量，包含这次添加的
result.list[i].products[j].pictures Array 商品图片列表：result.pictures[0]为50X50,result.pictures[1]为90X90,result.pictures[2]为120X120
